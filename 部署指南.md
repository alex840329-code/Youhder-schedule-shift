# 祐德牙醫排班系統 — 雲端部署指南
> 方案：Streamlit Community Cloud（免費）+ Supabase（免費）
> 完成後：你和合夥人可以在任何裝置用瀏覽器開啟，共同編輯，資料存在雲端。

---

## 總覽（整個流程約 30 分鐘）

```
本地程式碼
    ↓ 上傳
GitHub（免費私有倉庫）
    ↓ 連結
Streamlit Community Cloud（免費託管）
    ↓ 讀寫
Supabase（免費雲端資料庫）
```

---

## 第一步：建立 GitHub 倉庫

> GitHub 是存放程式碼的地方，Streamlit 從這裡讀取你的 App。

1. 前往 [github.com](https://github.com) → 登入（沒有帳號就免費註冊）
2. 右上角 **New** → 建立新倉庫
   - Repository name：`yude-schedule`（或你喜歡的名字）
   - **Private**（私有，合夥人無法看到原始碼，除非你邀請）
   - 勾選 **Add a README file**
   - 點 **Create repository**
3. 倉庫建立後，點 **Add file → Upload files**
4. 把整個排班程式資料夾的**所有檔案**拖曳上傳（除了 `.streamlit/secrets.toml` 和 `yude_config_v11.json`，這兩個不要上傳）
5. 點 **Commit changes**

**確認上傳的檔案清單（應包含）：**
```
app.py
requirements.txt
.gitignore
.streamlit/secrets.toml.example     ← 範例檔，可以上傳
modules/
pages/
```

## 第二步：建立 Supabase 資料庫

> Supabase 是免費的雲端 PostgreSQL 資料庫，用來儲存排班設定。

1. 前往 [supabase.com](https://supabase.com) → **Start your project**（用 GitHub 登入最方便）
2. **New project**
   - Organization：選預設
   - Name：`yude-clinic`
   - Database Password：設一個強密碼（記下來備用）
   - Region：**Northeast Asia (Tokyo)**（離台灣最近）
   - 點 **Create new project**（等約 1 分鐘）

3. 建立資料表（只需要一張）：
   - 左側選單 → **SQL Editor** → **New query**
   - 貼上以下 SQL 並點 **Run**：

```sql
-- 建立排班設定資料表
CREATE TABLE clinic_config (
    id      BIGINT PRIMARY KEY DEFAULT 1,
    data    JSONB NOT NULL DEFAULT '{}'::jsonb,
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 插入一筆空資料（讓程式第一次執行時有地方寫入）
INSERT INTO clinic_config (id, data) VALUES (1, '{}'::jsonb)
ON CONFLICT (id) DO NOTHING;

-- 讓 updated_at 自動更新
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER clinic_config_updated_at
BEFORE UPDATE ON clinic_config
FOR EACH ROW EXECUTE FUNCTION update_updated_at();
```

4. 取得連線金鑰：
   - 左側 → **Settings** → **API**
   - 複製 **Project URL**（形如 `https://xxxx.supabase.co`）
   - 複製 **anon / public** 金鑰（`eyJhbGci...` 開頭的長字串）

---

## 第三步：部署到 Streamlit Community Cloud

1. 前往 [share.streamlit.io](https://share.streamlit.io) → 用 GitHub 帳號登入
2. **New app**
   - Repository：選 `你的帳號/yude-schedule`
   - Branch：`main`
   - Main file path：`app.py`
3. 點 **Advanced settings** → **Secrets**，貼入：

```toml
SUPABASE_URL = "https://你的專案ID.supabase.co"
SUPABASE_KEY = "eyJhbGci...你的anon key..."
```

4. 點 **Deploy!**（首次部署約 2-3 分鐘）
5. 部署完成後會得到一個網址，例如：
   `https://yude-schedule-alex.streamlit.app`

## 第四步：邀請合夥人

**方法 A：直接分享網址（推薦）**
- 把 App 網址傳給合夥人，對方直接在瀏覽器開啟即可使用
- 資料即時同步（儲存後約 1 秒內對方重整就能看到最新資料）

**方法 B：設定密碼保護（選用）**
- 若不想讓外人誤入，可在 `app.py` 首頁加入簡單密碼驗證

## 常見問題

**Q：App 睡著了（顯示 "Zzz"），資料會不見嗎？**
A：不會。資料存在 Supabase，App 睡著只是暫停運算。點一下畫面喚醒即可（約 15-30 秒），資料完全保留。

**Q：兩個人同時編輯會有問題嗎？**
A：最後儲存的會覆蓋前一個人的版本。建議一次一人操作，或分工不同頁面（例如一人管人員設定，一人做排班）。

**Q：如何更新程式碼？**
A：修改後重新上傳到 GitHub，Streamlit Cloud 會在幾分鐘內自動偵測並重新部署。

**Q：費用是多少？**
A：Streamlit Community Cloud 免費（1 個 App）；Supabase 免費方案包含 500 MB 資料庫，排班系統的資料量遠低於此限制。

**Q：本地（電腦上）還能正常跑嗎？**
A：可以。本地執行時沒有 `secrets.toml` 就自動使用本地 JSON 存檔，不需要 Supabase。

## 本地開發環境設定（需要測試 Supabase 時）

若想在本地測試雲端存取，建立 `.streamlit/secrets.toml`（參考 `secrets.toml.example`）：

```toml
SUPABASE_URL = "https://xxxx.supabase.co"
SUPABASE_KEY = "eyJhbGci..."
```

然後執行：
```bash
pip install -r requirements.txt
streamlit run app.py
```

---

## 資料備份

App 每次儲存都會同時：
1. 寫入 Supabase（雲端主要資料）
2. 更新本地 `yude_config_v11.json`（本地備份）

若需要手動備份，在首頁（app.py）有「備份 / 匯入 JSON」功能，可以下載一份設定檔存到電腦。
